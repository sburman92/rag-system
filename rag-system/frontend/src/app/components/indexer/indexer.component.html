📖 How it works

  1. Read: The system clones and reads the GitHub repository
  2. Chunk: Code files are split into overlapping chunks (1000 chars per chunk)
  3. Embed: Each chunk is converted to embeddings using Ollama
  4. Store: Embeddings are stored in Chroma vector database
  5. Query: When you ask a question, the system finds similar chunks and sends them to LLM
